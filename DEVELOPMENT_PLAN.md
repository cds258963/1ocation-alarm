## 风险与应对

### 技术风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 后台被杀 | 高 | 高 | 多厂商适配测试，引导用户设置白名单 |
| 地理围栏不触发 | 中 | 高 | 使用高精度模式，增加定位频率 |
| 耗电过快 | 中 | 中 | 提供省电模式，优化定位策略 |
| 高德 SDK 兼容性问题 | 低 | 高 | 提前测试，准备备选方案 |

### 时间风险

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 环境搭建困难 | 中 | 中 | 详细记录安装步骤，准备 Docker 方案 |
| 功能复杂度超预期 | 中 | 中 | 优先保证 MVP，砍掉 P2/P3 功能 |
| 测试发现问题多 | 高 | 中 | 预留缓冲时间，快速迭代修复 |

## 代码规范

### Git 提交规范
```
feat: 新功能
fix: 修复 bug
docs: 文档更新
style: 代码格式
refactor: 重构
test: 测试
chore: 构建/工具
```

示例：
```bash
git commit -m "feat(alarm): 实现闹钟创建功能"
git commit -m "fix(geofence): 修复围栏触发不准确问题"
```

### 代码风格
- 遵循 [Effective Dart](https://dart.dev/guides/language/effective-dart)
- 使用 `flutter analyze` 检查代码
- 使用 `flutter format` 格式化代码
